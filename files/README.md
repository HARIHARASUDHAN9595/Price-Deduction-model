# 💻 Laptop Price Predictor

A machine learning web app that predicts the price of a laptop based on its specifications (brand, processor, RAM, storage, GPU, screen, OS, etc.). Built with **scikit-learn** for the model and **Streamlit** for the web interface.

## Demo

Enter laptop specs in the sidebar/form and get an instant predicted price in ₹ (INR).

## Dataset

- **Rows:** 1303 laptops
- **Columns:** Company, TypeName, Inches, ScreenResolution, Cpu, Ram, Memory, Gpu, OpSys, Weight, Price
- **Source:** Public laptop pricing dataset (Kaggle-style)

**Note:** The dataset is imbalanced across brands. Dell, Lenovo, and HP make up the majority of rows, while Apple has only ~21 rows (~1.6% of the data). This means predictions for Apple/macOS laptops are less reliable than for the well-represented brands — see Limitations below.

## Tech Stack

- Python, pandas, NumPy
- scikit-learn (ColumnTransformer, OneHotEncoder, SimpleImputer, RandomForestRegressor)
- Streamlit (web app)
- joblib (model serialization)

## Project Structure

```
├── model.py            # Trains the model and saves model.pkl
├── app.py               # Streamlit app that loads model.pkl and serves predictions
├── laptop_data.csv      # Training dataset
├── model.pkl             # Saved trained pipeline (generated by model.py)
└── requirements.txt     # Python dependencies
```

## Model Details

- **Algorithm:** RandomForestRegressor (n_estimators=200)
- **Preprocessing:** OneHotEncoder for categorical columns, median imputation for numerical columns, wrapped in a single sklearn `Pipeline`
- **Current R² Score:** 0.77 on the test split

## Known Limitations

- **Brand imbalance:** Apple has very few rows compared to Dell/Lenovo/HP, so predictions for Apple laptops are less accurate and the model may not generalize well to unseen Apple spec combinations.
- **High-cardinality text features:** Cpu, Gpu, Memory, and ScreenResolution are currently passed as raw free-text strings into a OneHotEncoder. Since most rows have a unique string (e.g. "Intel Core i5 7200U 2.5GHz"), the model treats nearly every laptop's spec as its own category, which limits how well it can generalize and is the main factor capping the R² score around 0.77.

## Planned Improvements

- Extract structured features instead of raw text — e.g. CPU brand + clock speed, GPU brand, storage type + size (GB) from Memory, and PPI (pixels per inch) from ScreenResolution.
- Use `stratify=Company` in `train_test_split` so each brand (including Apple) is proportionally represented in both train and test sets.
- Apply a log transform (`np.log1p`) to Price, since price distributions are typically right-skewed.
- Try Gradient Boosting / XGBoost and tune hyperparameters with GridSearchCV or RandomizedSearchCV.
- Collect more samples for underrepresented brands (Apple, Samsung, LG, etc.) to reduce bias.
